## 1. Qt升级发现的问题

01. 函数返回值类型问题,如：应返回指针，但返回了bool
02. QString直接与double相加
03. 非标准扩展。QVector及vector "swap trick"使用错误，应写为`vector<T>().swap(vec);`
04. 非标准扩展。非常量左值引用绑定临时右值问题，按需改成常量左值引用或普通变量
05. 字符串判断为空写成了字符串地址判断为空。(C4130)
06. 非标准扩展。函数形参为引用，但使用了临时右值。
07. 类函数声明中多加了限定名。(C4596)
08. for each in应改为c++ 11 range for形式
09. 函数形参为char*但实参为宏定义字符串(C2664)
10. qDebug最后不应为endl，应改为Qt::endl或不加
11. QFlags&lt;Qt::WindowType&gt;::QFlags构造时不应为int值，而应为枚举值
12. QWheelEvent中x()/y()应改为position().x()/y()；delta()应改为angleDelta.y()

## MSVC 编译器的警告 C4505

MSVC 编译器的警告 **C4505** ("unreferenced local function has been removed") 中文通常翻译为“已删除具有内部链接的未引用函数”。

简单来说，这个警告的意思是：**你写了一个只在当前文件内部可见的函数（通常是 `static` 函数），但是你在代码里从来没有调用过它，所以编译器把它删掉了。**

下面是关于这个警告的详细解释、产生原因以及解决方法：

### 1. 警告的具体含义

*   **具有内部链接 (Internal Linkage/Local Function):**
    在 C 或 C++ 中，如果你在函数定义前加上 `static` 关键字，这个函数就具有“内部链接”。这意味着它只能在定义它的那个 `.c` 或 `.cpp` 文件（编译单元）内部被看见和调用，其他文件无法访问它。

*   **未引用 (Unreferenced):**
    编译器在分析当前文件时，发现没有任何代码调用了这个函数，也没有任何指针指向这个函数的地址。

*   **已删除 (Removed):**
    既然这个函数只能给自己人用，而自己人又不用它，编译器为了节省生成的程序体积（EXE 或 DLL 的大小），决定在优化阶段直接把这个函数的机器码删掉，不编译进最终结果。

### 2. 为什么会出现这个警告？

通常有以下几种情况会导致 C4505：

01.  **代码残留 (Dead Code):**
    你之前写了一个辅助函数 `static void Helper() {...}` ，后来重构代码，把调用 `Helper()` 的地方删了或注释掉了，但是忘记把 `Helper` 函数本身的定义删掉。

02.  **头文件中的静态函数 (常见坑点):**
    你把一个 `static` 函数的**定义**（即函数体 `static void func() { ... }` ）写在了 `.h` 头文件里。

    -   **后果:** 任何包含这个头文件的 `.cpp` 文件，都会生成一份这个函数的副本。
    -   **触发:** 如果某个 `.cpp` 包含了头文件，但实际上没有用到这个函数，编译器就会针对这个 `.cpp` 报出 C4505 警告。
03.  **条件编译:**
    函数被宏包围了，例如：
    

```cpp
    static void DebugLog() { ... }

    void MainFunc() {
    #ifdef _DEBUG
        DebugLog(); // 只有在 Debug 模式下才调用
    #endif
    }
    ```

    如果你在 Release 模式下编译， `DebugLog()` 就没有被调用，从而触发警告。

### 3. 如何解决？

根据产生的原因，有不同的处理方式：

#### 方法 A：如果你确实不再需要这个函数

**直接删除它。** 这是清理代码、减少技术债的好机会。

#### 方法 B：如果你是误把 static 函数定义写在了头文件里

**不要在头文件中定义 static 函数。**
*   **做法:** 在头文件中只写声明（不要加 static），在某一个 `.cpp` 文件中写定义。
*   **或者:** 如果你想让函数像 `static` 一样每个文件都有一份且不冲突，应该使用 `inline` 关键字（C++ 中推荐），或者在 C++ 中使用匿名命名空间。

#### 方法 C：如果你暂时想保留它，或者是因为条件编译导致的

你可以使用 MSVC 的 `pragma` 宏来临时屏蔽这个警告，或者欺骗编译器说你用了它。

01.  **使用 `[[maybe_unused]]` (C++17 及以上推荐):**
    这是最现代、最优雅的方法。
    

```cpp
    [[maybe_unused]] static void MyFunc() {
        // ...
    }
    ```

02.  **宏屏蔽 (针对旧版本):**
    

```cpp
    #pragma warning(push)
    #pragma warning(disable: 4505)
    static void MyOldFunc() { ... }
    #pragma warning(pop)
    ```

### 总结

C4505 是一个**良性警告**（级别为 4，默认可能不显示，取决于你的警告等级设置）。它提示你有死代码。编译器是在帮你优化程序体积，但同时也提醒你：**“嘿，这有个函数你是不是忘了删，或者是写错地方了？”**
