《代码大全2》（Code Complete 2）中提到的**表驱动法（Table-Driven Method）**是一种非常优雅的编程技巧。它的核心思想是：**用查表来代替复杂的 `if-else` 或 `switch-case` 逻辑判断。**

在现代C++（C++11/14/17/20）中，得益于 `constexpr`、`std::array`、`std::unordered_map`、`std::function` 以及 Lambda 表达式的支持，表驱动法不仅可以写得极其简洁，还能在**编译期**完成求值，实现零运行期开销（Zero-overhead）。

《代码大全2》将表驱动法分为三种主要应用场景：**直接访问表**、**索引访问表**和**阶梯访问表**。下面我们用现代C++来一一演示。

---

### 1. 直接访问表 (Direct Access Table)

**场景**：输入是连续的整数（或可以转换为连续整数的枚举），直接作为数组的索引。
**现代C++利器**：`enum class` + `std::array` + `constexpr`

#### 示例 1A：数据查询（编译期查表）
获取某个月份的天数。过去你可能会写一个长长的 `switch-case`。现在我们用 `constexpr std::array`。

```cpp
#include <iostream>
#include <array>

// 强类型枚举，底层类型默认为 int
enum class Month { Jan, Feb, Mar, Apr, May, Jun, Jul, Aug, Sep, Oct, Nov, Dec, Count };

// 编译期常量表：由于使用了 constexpr，这个表存在于只读数据段，毫无运行期初始化开销
constexpr std::array<int, static_cast<size_t>(Month::Count)> DaysInMonth = {
    31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31
};

int getDays(Month m) {
    // 现代C++的安全边界检查（也可以用 [] 实现无检查的极速访问）
    return DaysInMonth.at(static_cast<size_t>(m)); 
}

int main() {
    std::cout << "Days in March: " << getDays(Month::Mar) << "\n";
    return 0;
}
```

#### 示例 1B：行为驱动（动作查表）
如果不同的状态需要执行不同的函数？我们可以将**函数指针**存入表中。

```cpp
#include <iostream>
#include <array>

enum class Event { Start, Pause, Stop, Count };

// 定义处理函数的类型
using EventHandler = void(*)();

void onStart() { std::cout << "Starting...\n"; }
void onPause() { std::cout << "Pausing...\n"; }
void onStop()  { std::cout << "Stopping...\n"; }

// 动作表：将枚举直接映射到函数指针
constexpr std::array<EventHandler, static_cast<size_t>(Event::Count)> ActionTable = {
    onStart, 
    onPause, 
    onStop
};

void handleEvent(Event e) {
    // 直接通过索引调用函数，O(1) 复杂度，替代了 switch-case
    ActionTable[static_cast<size_t>(e)]();
}
```

### 2. 索引访问表 (Indexed Access Table)

**场景**：输入条件不是连续的整数，比如字符串、稀疏的ID等。
**现代C++利器**：`std::unordered_map` + `std::function` + `Lambda 表达式`

#### 示例：命令路由器（Command Router）
处理不同字符串格式的命令。这种模式在编写 CLI 工具或服务器路由时非常常见。

```cpp
#include <iostream>
#include <string>
#include <unordered_map>
#include <functional>

class CommandProcessor {
    // 使用 std::function 允许我们存入普通的函数、成员函数、甚至是带捕获的 Lambda
    using CommandAction = std::function<void(const std::string&)>;

    // 哈希表：建立 字符串 -> 动作 的映射
    std::unordered_map<std::string, CommandAction> commandTable_;

public:
    CommandProcessor() {
        // 在构造时注册表项（结合了 Lambda 表达式，代码非常内聚）
        commandTable_ = {
            {"help",  [](const auto& args) { std::cout << "Show Help\n"; }},
            {"print", [](const auto& args) { std::cout << "Print: " << args << "\n"; }},
            {"exit",  [](const auto& args) { std::cout << "Exiting...\n"; }}
        };
    }

    void execute(const std::string& cmd, const std::string& args = "") {
        // 查表
        if (auto it = commandTable_.find(cmd); it != commandTable_.end()) {
            it->second(args); // 执行对应的 Lambda
        } else {
            std::cout << "Unknown command: " << cmd << "\n";
        }
    }
};

int main() {
    CommandProcessor processor;
    processor.execute("print", "Hello Table-Driven");
    processor.execute("help");
    processor.execute("jump"); // Unknown
    return 0;
}
```

### 3. 阶梯访问表 (Stair-Step Access Table)

**场景**：输入是一个范围，而不是具体的值。比如：分数转换成等级（90-100为A，80-89为B...）。
**现代C++利器**：`struct` + `std::array`/`std::vector` + `std::ranges::find_if` (C++20)

在这个场景中，使用 `if (score >= 90) ... else if (score >= 80) ...` 很容易出错（顺序写反就会导致 bug）。表驱动法将**逻辑与数据解耦**。

```cpp
#include <iostream>
#include <array>
#include <algorithm>

// 定义阶梯的结构：上限阈值和对应的结果
struct GradeBoundary {
    double upperLimit;
    std::string_view grade; // C++17 字符串视图，轻量且高效
};

// 必须按 upperLimit 从小到大排序（阶梯）
constexpr std::array<GradeBoundary, 5> GradeTable = {{
    { 59.9, "F (Fail)" },
    { 69.9, "D" },
    { 79.9, "C" },
    { 89.9, "B" },
    { 100.0, "A" }
}};

std::string_view getGrade(double score) {
    if (score < 0.0 || score > 100.0) return "Invalid";

    // C++20 std::ranges 查找第一个上限大于等于当前分数的阶梯
    auto it = std::ranges::find_if(GradeTable, [score](const GradeBoundary& boundary) {
        return score <= boundary.upperLimit;
    });

    if (it != GradeTable.end()) {
        return it->grade;
    }
    return "Invalid";
}

int main() {
    std::cout << "Score 85.5 is " << getGrade(85.5) << "\n";
    std::cout << "Score 50.0 is " << getGrade(50.0) << "\n";
    std::cout << "Score 99.9 is " << getGrade(99.9) << "\n";
    return 0;
}
```

### 总结：现代C++ 为表驱动法带来的提升

相比于《代码大全2》成书时期（主要针对 C 或早期 C++98/03），现代 C++ 让表驱动法变得更加完美：

1. **零运行时开销 (`constexpr`)**：直接访问表和阶梯访问表都可以声明为 `constexpr`，查表动作在编译期就可以完成优化，甚至不需要在运行时分配内存。
2. **极高的表达力 (Lambda & `std::function`)**：以前在 C 语言中做行为表，只能传函数指针，无法携带状态。现在可以直接在表里塞入带捕获的 Lambda，实现闭包机制。
3. **类型安全 (`enum class` & 结构化绑定)**：消除了传统宏定义和弱类型 `enum` 越界带来的风险。
4. **解耦逻辑与数据**：日后增加新的命令、新的事件、修改分数线，**都不需要修改核心业务代码，只需要在初始化大括号 `{}` 里增加一行数据即可**。符合面向对象中的“开闭原则（OCP）”。
